Introduction

SQL (Structured Query Language) is a domain-specific language used in programming and designed for managing data held in a relational database management system (RDBMS). It is particularly useful for handling structured data where there are relations between different entities and variables.

SQL contains a standard library of commands, such as SELECT, INSERT, UPDATE, DELETE, and a core set of language elements such as operators, control structures, and statements. Core SQL can be extended for a variety of purposes by supplementing it with additional commands; for example:

  • Data Query Language (DQL): Commands like SELECT are used to fetch data from a database.
  • Data Definition Language (DDL): Commands like CREATE, ALTER, and DROP are used to define or alter database schema.
  • Data Control Language (DCL): Commands like GRANT and REVOKE are used to control access to data.
  • Data Manipulation Language (DML): Commands like INSERT, UPDATE, and DELETE are used to manipulate data within schema objects.
What you should already know

This guide assumes you have the following basic background:

  • A general understanding of databases and their role in storing and managing data.
  • Basic familiarity with programming concepts and terminology.
  • Some experience with data manipulation or querying languages. If you are new to SQL, try one of the introductory tutorials linked on the main page about SQL.
SQL and Other Query Languages

SQL is often compared to other data manipulation and query languages but stands out due to its widespread use and standardization. While other languages like NoSQL are used for specific use cases such as handling unstructured data, SQL remains the preferred choice for relational databases because of its robust query capabilities and well-defined standards.

In contrast to some other query languages that might require different methods for different types of data stores, SQL provides a unified language for managing relational data. It supports operations like data querying, schema creation and modification, data access control, and data manipulation within a single, consistent framework.

First Query
To get started with writing SQL, you can execute your first "Hello World" SQL query by connecting to your database and running the following command to fetch all data from a table named 'users': SELECT * FROM users; This command retrieves all columns from all rows in the 'users' table.
Variables

SQL uses variables to store temporary data during the execution of a SQL query or procedure. Variables are often used in SQL scripts and stored procedures to store data that can be reused multiple times.

Declaring Variables

You can declare a variable in SQL using the DECLARE keyword. For example: DECLARE @myVariable INT; SET @myVariable = 10; In this example, a variable named @myVariable of type INT is declared and then set to a value of 10.

Data Types
SQL supports a variety of data types, which can be categorized into several groups:
  • Numeric types: INT, FLOAT, DECIMAL, etc.
  • Character types: CHAR, VARCHAR, TEXT, etc.
  • Date and time types: DATE, TIME, DATETIME, TIMESTAMP, etc.
  • Binary types: BINARY, VARBINARY, BLOB, etc.
  • Boolean type: BIT

These data types enable you to define the kind of data that can be stored in each column of a table, ensuring data integrity and efficient storage.

Control Structures

SQL provides various control structures to manage the flow of execution within scripts and stored procedures, including conditional statements and loops.

IF...ELSE Statement

The IF...ELSE statement is used to execute a set of SQL statements based on a condition. The syntax is as follows:

IF condition BEGIN -- statements to execute if condition is true END ELSE BEGIN -- statements to execute if condition is false END

Example:

IF @myVariable = 10 BEGIN PRINT 'Variable is 10'; END ELSE BEGIN PRINT 'Variable is not 10'; END

WHILE Statement

The WHILE statement is used to repeatedly execute a block of SQL statements as long as a specified condition is true. The syntax is as follows:

WHILE condition BEGIN -- statements to execute as long as condition is true END

Example:

DECLARE @counter INT = 0; WHILE @counter < 5 BEGIN PRINT 'Counter is ' + CAST(@counter AS NVARCHAR(10)); SET @counter = @counter + 1; END
Functions

SQL allows you to create user-defined functions to encapsulate reusable logic. A function in SQL is defined with the CREATE FUNCTION statement, specifying the function name, input parameters, return type, and the function body.

Example of a simple scalar function that returns the square of a number:

CREATE FUNCTION dbo.Square (@number INT) RETURNS INT AS BEGIN RETURN @number * @number; END;

To call this function:

SELECT dbo.Square(5);
